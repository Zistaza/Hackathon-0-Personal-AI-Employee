# Email Approval Workflow - Human-in-the-Loop

Enhanced orchestrator with Human-in-the-Loop workflow for email execution.

## Overview

The email approval workflow ensures that NO email is sent automatically. Every email requires explicit human approval before execution.

## Architecture

```
Email Action Detected
    ↓
Create Approval Request → /Pending_Approval/email/
    ↓
Wait for Human Decision
    ↓
    ├─→ Human Moves to /Approved/
    │       ↓
    │   Send Email via MCP
    │       ↓
    │   Move to /Done/
    │       ↓
    │   Log to /Logs/YYYY-MM-DD.json
    │
    └─→ Human Moves to /Rejected/
            ↓
        Log Rejection
            ↓
        No Email Sent
```

## Directory Structure

```
AI_Employee_Vault/
├── Pending_Approval/
│   └── email/              # Email approval requests wait here
├── Approved/               # Human moves approved emails here
├── Rejected/               # Human moves rejected emails here
├── Done/                   # Successfully sent emails archived here
└── Logs/
    └── YYYY-MM-DD.json    # Daily email action logs
```

## Approval File Format

Each email approval request is a markdown file with this structure:

```markdown
---
action: send_email
to: recipient@example.com
subject: Meeting Reminder
timestamp: 2026-02-24T10:30:00.000Z
status: pending
---

body:
Don't forget our meeting tomorrow at 10 AM.

Please review the attached documents before the meeting.

Best regards,
AI Employee

---

## Instructions for Human Approval

To APPROVE this email:
1. Review the email details above
2. Move this file to: /Approved/
3. The orchestrator will automatically send the email

To REJECT this email:
1. Move this file to: /Rejected/
2. The orchestrator will log the rejection

DO NOT edit this file. Simply move it to the appropriate folder.
```

## Workflow Steps

### Step 1: Email Action Detected

When your AI Employee system needs to send an email, it creates an approval request:

```python
from create_email_approval import create_email_approval

create_email_approval(
    to="client@example.com",
    subject="Project Update",
    body="The project is on track for delivery next week."
)
```

This creates a file in `/Pending_Approval/email/`

### Step 2: Orchestrator Detects Request

The orchestrator watches `/Pending_Approval/email/` and logs:

```
[2026-02-24 10:30:00] [INFO] Email approval request created: email_approval_20260224_103000.md
[2026-02-24 10:30:00] [INFO] Waiting for human to move file to /Approved/ or /Rejected/
```

**IMPORTANT:** The orchestrator does NOTHING at this stage. It waits for human action.

### Step 3: Human Reviews and Decides

**Option A: Approve**
1. Open the file in `/Pending_Approval/email/`
2. Review the recipient, subject, and body
3. Move (or cut/paste) the file to `/Approved/`

**Option B: Reject**
1. Open the file in `/Pending_Approval/email/`
2. Decide not to send
3. Move (or cut/paste) the file to `/Rejected/`

### Step 4A: Approved - Email Sent

When file appears in `/Approved/`:

```
[2026-02-24 10:35:00] [INFO] Processing approved email: email_approval_20260224_103000.md
[2026-02-24 10:35:00] [INFO] Sending email to: client@example.com
[2026-02-24 10:35:01] [INFO] Email sent successfully
[2026-02-24 10:35:01] [INFO] Moved to Done: sent_20260224_103501_email_approval_20260224_103000.md
```

The orchestrator:
1. Reads the approval file
2. Extracts email data (to, subject, body)
3. Sends email via Email MCP server
4. Logs action to `/Logs/2026-02-24.json`
5. Moves file to `/Done/sent_TIMESTAMP_filename.md`
6. Marks approval as processed in `processed_approvals.json`

### Step 4B: Rejected - Email Cancelled

When file appears in `/Rejected/`:

```
[2026-02-24 10:35:00] [INFO] Email approval rejected: email_approval_20260224_103000.md
[2026-02-24 10:35:00] [INFO] Rejected email to: client@example.com
[2026-02-24 10:35:00] [INFO] Subject: Project Update
```

The orchestrator:
1. Logs rejection to `/Logs/2026-02-24.json`
2. File remains in `/Rejected/` for your records
3. NO email is sent

## Duplicate Prevention

The orchestrator tracks processed approvals in `processed_approvals.json`:

```json
{
  "abc123def456": {
    "filename": "email_approval_20260224_103000.md",
    "to": "client@example.com",
    "subject": "Project Update",
    "status": "sent",
    "processed_at": "2026-02-24T10:35:01Z"
  }
}
```

This prevents:
- Sending the same email twice if file is moved multiple times
- Processing duplicate approval requests
- Accidental re-execution

## Logging

All email actions are logged to `/Logs/YYYY-MM-DD.json`:

```json
[
  {
    "timestamp": "2026-02-24T10:35:01Z",
    "action": "email_sent",
    "to": "client@example.com",
    "subject": "Project Update",
    "details": "250 2.0.0 OK  1771884666 98e67ed59e1d1-358d7a5c70csm179286a91.5 - gsmtp"
  },
  {
    "timestamp": "2026-02-24T11:20:00Z",
    "action": "email_rejected",
    "to": "vendor@example.com",
    "subject": "Payment Request",
    "details": "User rejected approval for email_approval_20260224_112000.md"
  }
]
```

## Usage Examples

### Example 1: Create Approval Request via CLI

```bash
cd Skills/integration_orchestrator

python3 create_email_approval.py \
  "client@example.com" \
  "Project Status Update" \
  "The project is progressing well. We expect to deliver by Friday."
```

Output:
```
✓ Email approval request created: /Pending_Approval/email/email_approval_20260224_103000.md
  To: client@example.com
  Subject: Project Status Update

Waiting for human approval...
  Move to /Approved/ to send
  Move to /Rejected/ to cancel
```

### Example 2: Create Approval Request Programmatically

```python
from pathlib import Path
import sys
sys.path.append('Skills/integration_orchestrator')
from create_email_approval import create_email_approval

# Create approval request
filepath = create_email_approval(
    to="team@company.com",
    subject="Weekly Report",
    body="This week we completed 5 tasks and started 3 new initiatives."
)

print(f"Approval request created: {filepath}")
```

### Example 3: Batch Approval Requests

```python
emails = [
    {
        "to": "client1@example.com",
        "subject": "Invoice #001",
        "body": "Please find attached invoice #001 for services rendered."
    },
    {
        "to": "client2@example.com",
        "subject": "Invoice #002",
        "body": "Please find attached invoice #002 for services rendered."
    }
]

for email in emails:
    create_email_approval(
        to=email["to"],
        subject=email["subject"],
        body=email["body"]
    )
```

## Troubleshooting

### Email Not Sending

**Check:**
1. Orchestrator is running: `ps aux | grep index.py`
2. File was moved to `/Approved/` (not copied)
3. Email MCP server is configured: `ls -la ../../mcp_servers/email_mcp/`
4. SMTP credentials are set: `cat ../../mcp_servers/email_mcp/.env`
5. Check logs: `tail -f ../../Logs/integration_orchestrator.log`

### Approval File Not Detected

**Check:**
1. File is `.md` format
2. File is in correct directory
3. Orchestrator is watching the directory
4. File wasn't already processed: `cat processed_approvals.json`

### Duplicate Email Sent

**This should NOT happen** due to duplicate prevention. If it does:
1. Check `processed_approvals.json` for the approval ID
2. Check logs for duplicate processing
3. Report as bug

## Security Considerations

### Best Practices

1. **Review Every Email:** Always read the full email before approving
2. **Check Recipients:** Verify email addresses are correct
3. **Sensitive Data:** Ensure no sensitive data in email body
4. **Backup Approvals:** Keep `/Done/` and `/Rejected/` for audit trail
5. **Monitor Logs:** Regularly review `/Logs/YYYY-MM-DD.json`

### Access Control

- Ensure only authorized users can access `/Approved/` directory
- Protect SMTP credentials in email MCP `.env` file
- Review orchestrator logs for unauthorized access attempts

## Maintenance

### Clean Up Old Files

```bash
# Archive old Done files (older than 30 days)
find ../../Done/ -name "sent_*.md" -mtime +30 -exec mv {} ../../Archive/ \;

# Clean up old Rejected files
find ../../Rejected/ -name "email_*.md" -mtime +30 -exec rm {} \;
```

### Reset State (if needed)

```bash
# Backup first
cp processed_approvals.json processed_approvals.json.backup

# Clear state
echo "{}" > processed_approvals.json
```
